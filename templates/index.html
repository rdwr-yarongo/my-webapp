Global Site Load Balancing Demonstration

In this section we will demonstrate global site load balancing.

Please note: the Main site has two links, and the DR site has one link:

  • Site A ISP link 1 is represented by VIP 10.100.4.54
  • Site A ISP link 2 is represented by VIP 10.100.5.54
  • Site B ISP link is represented by VIP 10.100.7.103

The Execute action performs multiple attempts to app1.radware.lab. Each attempt triggers a DNS decision and then sends HTTP traffic to the selected destination so you can observe GSLB behavior over time.

1) DNS Query

Flow: Controller asks DNS for app1.radware.lab, DNS returns one destination per attempt based on GSLB policy, then controller sends HTTP to that selected target.
