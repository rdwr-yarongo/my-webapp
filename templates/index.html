Please note: the Main site has two links, and the DR site has one link:

HA Failover

Run HA Failover

Monitor live traffic for app2.radware.lab, trigger a failover from Alteon 1 to Alteon 2, and then restore the primary unit.

High Availability Demonstration

Alteon 1 starts as the master ADC and Alteon 2 starts as the backup ADC in the HA pair.

The failover action forces Alteon 1 to relinquish traffic by shutting down ports 1, 2, and 3 over the Alteon API.

The restore action brings the same interfaces back up so the primary unit can return to service.

In the live results pane, the customer can watch the embedded app2.radware.lab page continue to load while the proof fields show which system handled the request after failover.

HTTP to HTTPS Demonstration

This scenario demonstrates HTTP redirection to HTTPS for scenario2.radware.lab.

The embedded browser frame starts from http://scenario2.radware.lab/ and then updates to https://scenario2.radware.lab/ after the Alteon redirect is confirmed.

The new-tab action remains available when you want the customer to see the real browser address bar perform the redirect naturally.

Execution Results Experience

When you click Launch Embedded Demo, the browser-style frame opens in the Execution Results pane on the right, consistent with the other scenarios.

The results pane will show the visible URL transition, the redirect proof, and the final HTTPS page.
