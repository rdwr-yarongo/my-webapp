HA Failover

Run HA Failover

Monitor live traffic for app2.radware.lab, trigger a failover from Alteon 1 to Alteon 2, and then restore the primary unit.

Alteon 1 starts as the master ADC and Alteon 2 starts as the backup ADC in the HA pair.

The failover action forces Alteon 1 to relinquish traffic by shutting down ports 1, 2, and 3 over the Alteon API.

The restore action brings the same interfaces back up so the primary unit can return to service.

In the live results pane, the customer can watch the embedded app2.radware.lab page continue to load while the proof fields show which system handled the request after failover.

Scenario 2 — Content Offloading

What is Content Offloading?

Alteon can modify HTTP traffic in transit: removing sensitive headers from responses, stripping strings from the body, and injecting new content — all without touching the backend server.

Click Load Page via Alteon to fetch https://scenario2.radware.lab/index.php through Alteon and view the page in the Execution Results pane.

Scenario 3 — Content-Switching

What is Content-Switching?

Alteon inspects each incoming request and routes it to a different backend server group based on the HTTP Host header. This allows a single VIP (10.100.4.56) to serve multiple environments simultaneously:

  • scenario3-dev.radware.lab → Development backend
  • scenario3-stg.radware.lab → Staging backend
  • scenario3.radware.lab → Production backend

Click any button below to send a real request through the Alteon VIP with the corresponding Host header and view the page served by that environment in the Execution Results pane.

Scenario 4 — HTTP/2 Gateway

What is HTTP/2 Gateway?

Alteon acts as an HTTP/2 termination gateway. The client sends requests over HTTP/2 (multiplexed, binary, header-compressed), while Alteon communicates with backend servers over HTTP/1.1. This provides modern protocol benefits to clients without requiring backend changes.

  • Client → HTTP/2 (TLS) → Alteon
  • Alteon → HTTP/1.1 → Backend servers
  • VIP: scenario4.radware.lab
